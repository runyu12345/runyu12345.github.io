面试问题列表

2018-05-07

前几天看到有人分享面试题,自己问了自己一下.发现有些问题是工作中经常用到的却没有系统的学习过,不能回答的很彻底.有些问题根本不知道.当然,也有些问题是研究过的,不过不是很系统.所以整理到博客,后续会抽空每个问题都附上一篇解答.

Java基础

  1. String为什么是final的?
  2. HashMap源码,实现原理,底层结构.
  3. 反射中,Class.forName和classloader的区别.
  4. session与cookie的区别和联系.session声明周期,多服务器部署时session管理.
  5. Java中的队列有哪些?有什么区别.
  6. Java的内存模型以及GC算法.
  7. Java7和Java8的新特性.
  8. Java数组和链表两种结构的操作效率,在哪些情况下(从头开始,从结尾开始,从中间开始),哪些操作(插入,查找,删除)的效率高.
  9. Java内存泄漏的问题调查定位,jmap,jstack的使用等.

框架

  1. struts2和SpringMVC的区别.
  2. Spring框架中需要引入哪些jar包,以及这些jar包的用途?(这是个什么鬼题目)
  3. SpringMVC的原理.注解的意思.
  4. Spring中BeanFactory和applicationContext的区别和联系.
  5. Spring注入的几种方式.
  6. Spring如何实现事务管理.
  7. SpringIOC和AOP的原理.
  8. Spring循环注入的方式.

多线程

  1. Java创建线程之后,直接调用start()方法和run()方法的区别.
  2. 常用的线程池模式以及不同线程池的使用场景.
  3. newFixedThreadPool此种线程池如果线程数达到最大值后会怎么办?底层原理.
  4. 多线程之间通信的同步问题,synchronized锁的是对象,延伸出和synchronized相关很多的具体问题,例如:
    同一个类不同方法都有synchronized锁,一个对象是否可以同时访问. 或者一个类的static构造方法加上synchronized之后的锁的影响.
  5. 可重入锁的含义,以及ReentrantLock和synchronized的区别.
  6. 同步的数据结构(即:线程安全的数据结构)例如:concurrentHashMap的源码理解以及内部实现原理,为什么他是同步的且效率高.
  7. atomicinteger和volatile等线程安全操作关键字的理解和使用.
  8. 线程间通信,wait和notify.
  9. 定时线程的使用.
  10. 场景:在一个主线程中,要求有大量的子线程执行完成之后,主线程才执行完成.多种方式,考虑效率.

网络通信

  1. HTTP是无状态通信,HTTP请求方式有哪些?可以自己定义新的请求方式么?
  2. socket通信,以及长连接,分包,连接异常断开的处理.
  3. socket通信模型的使用, AIO和NIO.
  4. socket框架Netty的使用,以及NIO的实现原理,为什么是异步非阻塞.
  5. 同步和异步,阻塞和非阻塞.

Linux

  1. 常用的Linux下的命令.
  2. 大的log文件中,统计异常出现的次数,排序,或者指定输出多少行多少列的内容.(awk)
  3. 调查问题的思路: 内存,CPU,句柄数,过滤,查找,模拟post和get请求等场景.
  4. shell脚本中 #!的作用.

数据库

  1. MySQL存储引擎的不同.(这个问题不做考虑,pgSQL的存储引擎做重点)
  2. 单个索引,联合索引,主键索引.
  3. 怎么分表,以及分表以后如果想按条件分页查询怎么办.
  4. 分表之后,想让一个id多个表是自增的,效率和实现.
  5. 实时备份同步的配置以及原理.读写分离.
  6. 索引的数据结构,B+树.
  7. 事务的四个特性. 项目怎么解决这些问题.
  8. 范式.

设计模式

  1. 单例模式(两种),饿汉模式的延迟加载.
  2. 工厂模式.
  3. 装饰着模式.
  4. 观察者模式.

缓存

  1. 为什么用缓存,用过哪些缓存,Redis和memcahe的区别. (公司项目用SSDB较多,所以 Redis和SSDB的区别也要了解)
  2. Redis的数据结构.
  3. Redis的持久化方式,以及项目中用的哪种,为什么?
  4. redis集群的理解,怎么动态增加或者删除一个节点.而保证数据不丢失.(一致性哈希)
